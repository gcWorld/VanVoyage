## Color Coding

### Warning Cards (Orange)
- Background: Light orange (`Colors.orange.shade50`)
- Text/Icon: Dark orange (`Colors.orange.shade700`)
- Icon: `Icons.warning_amber_outlined`
- Use: Values outside recommended range but acceptable

### Error Cards (Red)
- Background: Error container color (theme-based red)
- Text/Icon: Error color (theme-based dark red)
- Icon: `Icons.error_outline`
- Use: Values outside safe limits

### Info Cards (Blue) - Currently not used
- Background: Primary container color (theme-based blue)
- Text/Icon: Primary color (theme-based dark blue)
- Icon: `Icons.info_outline`
- Use: Informational messages

## Dynamic Behavior

1. **Immediate Display**: Warnings appear as soon as user moves a slider
2. **Real-time Updates**: Warning text updates based on current values
3. **Automatic Removal**: Warnings disappear when values return to valid range
4. **Section Visibility**: "Travel Constraint Warnings" section only shows when violations exist
5. **Multiple Warnings**: All applicable warnings shown simultaneously
6. **Scrollable**: Warning cards are part of the scrollable form

## User Experience Flow

1. User opens Travel Constraints form
2. User adjusts Max Daily Distance slider to 200 km (below recommended)
3. Orange warning card appears immediately below the form
4. User continues to adjust - warning updates in real-time
5. User moves slider to 400 km (within recommended range)
6. Warning card disappears
7. User moves slider to 1100 km (above maximum)
8. Red error card appears
9. User can still save preferences despite warnings

## Accessibility Considerations

- Color is not the only indicator (icons used)
- Clear text descriptions of issues
- Expected values always shown
- Severity levels clearly distinguished
- Screen reader compatible (semantic HTML/widgets)
- Keyboard navigable

## Implementation Notes

- Warnings are calculated using `TravelConstraintValidator`
- Validation runs on every slider/switch change
- UI uses Material 3 design system colors
- Cards use theme-aware colors for light/dark mode
- No network calls or async operations needed
- All validation is local and instant
